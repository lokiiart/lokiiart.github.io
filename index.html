封心锁爱之关机篇

马上又要开始备考了，最近这段时间沉迷于计算机的回炉重修，实实在在
的收获不小，有时候甚至忍不住感慨，真得是走了不少弯路。

 唏嘘的话也不想多说了，从现在开始，打下坚实的基础，还是来得及的，
 还是那句老话，种一棵树最好的时候是十年前，其次是现在。现在我下决
 心重新打基础，凭着这十来年摸爬滚打的经验，我相信，超越自己是不需
 要花费太多时间的，甚至得到自己的认可信心百倍地在程序这一块直接出
 击也可能不需要多少时间。程序员这个职业现在火成这个样子，我早已心
 急如焚，可我没实力啊，自己都不认可啊，怎么去蒙别人呢。

 但是现在，真的，虽然只有不长的时间，我从C语言入手，虽然没有做出什
 么实用的应用，我却已觉得比我之前已经走得要远了。这也是由我自己的
 性格决定的，说到底我其实是较真的性格，不是说我能完成工作任务就可
 以的。我给自己的定位就是要成为大神，自然有的时候就眼高手低了，可
 我不觉得可耻，因我这些年来从来没放弃过。

 C语言确实是一门非常好用也非常好学的语言，只是在最开始需要花费一点
 点耐心，我简单概括一下我最近所得。

 1. 面向对象的方式使用C语言。
1. 类（完成）
2. 封装
3. 继承
4. 多态
 2. TDD学习大法---直接抄代码很难说服自己有所得，看一遍需求自己动笔又
    不知从哪下手，所以我独创了TDD学习大法，这法子当然不是我原创，
    但用于学习我倒是没见过。
1. 简单的测试框架（完成-minunit.h)
2. 简单的测试流程（驱动代码driver，代码桩stub，mock代码）已完成
3. 在学习中使用TDD学习大法
4. 在项目学习中中使用TDD大法
 3. 完成Vim的配置
 4. 应用了Emacs的OrgMode


 然后再简单说下下一步的学习计划，我现在以为计算机的学习需遵循以下几个原则：

 1. 同一时期应该要熟悉三门语言，一门用于开发实际项目（现阶段是
    Python），一门用于学习（C语言），一门用于编写各种工具（Shell）。
 2. 计算机的学习应该做好文理分工，文科是指编程语言，项目架构（自动
    布署测试等），设计模式等，理科是指数据结构，操作系统原理，网络
    协议，编译原理等。
 3. 坚持应用TDD学习大法，尤其是学习的时候，对理解代码太有用了。


 具体学习流程安排如下：

 1. 数据结构C语言版
 2. 《C++程序设计原理与实践》
1. 四则运算计算器的实现与抽象思想
2. 面向过程的C与面向对象的C++在程序设计上的异同
 3. 数据结构（C++版）
 4. 操作系统原理（清华ucore实践）
 5. iOS编程+iOS设计模式
 6. 错题本项目起步（基本功能-绘图、分类等）
 7. 算法+AI
 8. 完成错题本项目上架


 这个阶段学得不错，希望你回来捡起来的时候更有信心和状态。请从
 github上的c-skeleton开始，最后，备考顺利。
